# Contributing to Drop

Thank you for your interest in contributing to Drop! This document provides guidelines for contributing.

## Code of Conduct

- Be respectful and inclusive
- Welcome newcomers
- Focus on constructive feedback
- Respect different viewpoints

## Getting Started

1. Fork the repository
2. Clone your fork
3. Install dependencies: `npm install`
4. Create a branch for your changes
5. Make your changes
6. Test your changes
7. Submit a pull request

## Development Setup

See the main [README.md](../README.md) for setup instructions.

## Coding Standards

### TypeScript

- Use TypeScript for all new code
- Follow the existing code style
- Use meaningful variable names
- Add comments for complex logic

### React

- Use functional components with hooks
- Keep components focused and small
- Use TypeScript for props

### Styling

- Use Tailwind CSS for styling
- Follow the brand guidelines (Orange #FD5E0F as accent only)
- Ensure dark mode support

## Pull Request Process

1. Update documentation if needed
2. Add tests if applicable
3. Ensure all tests pass
4. Update CHANGELOG.md if applicable
5. Submit PR with clear description

## Commit Messages

Use clear, descriptive commit messages:

```
feat: Add password protection for file shares
fix: Resolve download issue with large files
docs: Update API documentation
```

## Testing

- Write tests for new features
- Ensure existing tests pass
- Test in multiple browsers
- Test with different file types and sizes

## Security

- Never commit secrets or API keys
- Report security issues privately
- Follow security best practices
- Review security implications of changes

## Questions?

Feel free to open an issue for questions or discussions.
